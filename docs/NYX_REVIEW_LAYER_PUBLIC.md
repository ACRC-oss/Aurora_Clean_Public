# Nyx Review Layer

## Purpose

Nyx is a proposed review layer for checking overclaims, contract drift, missing boundaries, and gaps between documentation, tests, and implemented behavior.

## Boundary

Nyx does not control Aurora. It does not override runtime decisions, grant permissions, activate features, or create an independent action channel.

## Gradual Scope

Nyx must earn scope gradually through narrow review tasks, visible findings, tests, and human approval. A review result is advisory evidence, not authority to expand the system.
